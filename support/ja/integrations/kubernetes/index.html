CircleCI で Kubernetes デプロイを自動化

CircleCI は、Kubernetes デプロイ管理の柔軟性、制御性、効率性を向上させます。

GitHub アカウントで登録
  • すべてのリポジトリ
  • パブリック リポジトリのみ
Bitbucket アカウントで登録

ユーザー登録を行うことで、当社の利用規約およびプライバシー ポリシーに同意したものと見なされます。CircleCI では、サービス改善を目的として、お客様のアカウント データへの読み取りアクセスまたは書き込みアクセスの許可をお願いすることがあります。

CircleCI と Kubernetes の連携で効率的な CI/CD パイプライン構築

Kubernetes と CircleCI でマイクロサービスを管理しましょう

  • Docker のネイティブ サポート

    CircleCI は Docker を徹底的にサポートしており、Kubernetes の本番環境のミラー環境でマイクロサービス アプリケーションをビルド、テストできます。

  • コンテナ レジストリのインテグレーション

    コンテナ イメージを保存、管理、保護するコマンドを CircleCI ジョブから直接実行できます。

  • デプロイ サービス

    CircleCI は、サードパーティのデプロイメントおよびクラスタ管理ツールと連携するため、コミットからデプロイまでのソフトウェアの継続的デリバリー パイプラインを構成できます。

CircleCI と Kubernetes の連携

当社パートナーが作成した Orbs を利用するだけで、CircleCI パイプラインと Kubernetes を連携できます。

カテゴリ
  1. Kubernetes マネージド サービス
  2. デプロイ
  3. レジストリ

Kubernetes マネージド サービス

Google Kubernetes Engine

circleci/gcp-gke

Kubernetes クラスタのデプロイや本番コードの更新を、わずか数秒で CI パイプラインから行うことができます。

Amazon EKS

circleci/aws-eks

AWS 上で Kubernetes を使用してコンテナ化されたアプリケーションのデプロイ、管理、スケーリングを行います。

Azure Kubernetes Service

circleci/azure-aks

Azure Kubernetes Service を使用して、リリース期間の短縮、運用の効率化、スケーリングの信頼性確保を実現できます。

Red Hat OpenShift

circleci/redhat-openshift

Kubernetes アプリケーションのビルド、デプロイ、管理を自動化します。

Kublr

kublr/kublr-api

大規模企業のあらゆる環境に含まれるマルチクラスタ Kubernetes デプロイメントについて、デプロイ、実行、管理を一元的に行います。

Helm

circleci/helm

Helm Charts を使用して、Kubernetes 向けに構築されたソフトウェアを検索、共有、使用することができます。

Nirmata

nirmata/nirmata

Nirmata API を使用し、開発、テスト、ステージング、本番の複数環境をまたいで Kubernetes アプリケーションを管理することができます。

DeployHub

deployhub/deployhub-orb

ワークフローのリリース ステップを自動化し、パイプラインの状態それぞれについて、100% 繰り返し実行可能なデプロイ プロセスを作成します。

VMware Code Stream

vmware/codestream

運用リスクを軽減しながら、高品質のアプリケーションや IT コードをすばやくリリースします。

Amazon ECR

circleci/aws-ecr

Amazon を利用してコンテナ イメージの保存、管理、デプロイを容易化します。

Google Container Registry

circleci/gcp-gcr

Google Container Registry を利用して、Docker コンテナ イメージを保存および管理し、セキュアに保ちます。

Docker Hub

circleci/docker

チームのコンテナ アプリケーションをどこからでも作成、管理、ビルド、リリースすることができます。

Azure Container Registry

circleci/azure-acr

Azure にデプロイするコンテナ イメージの保存と管理を容易に行えるため、Kubernetes コンテナ開発を簡素化できます。

よく寄せられるご質問

Kubernetes とは?

Kubernetes は、コンテナ化されたワークロードやサービスを管理するための移植可能で拡張性のあるオープンソース プラットフォームであり、宣言型の構成管理と自動化を促進します。コンテナ中心の管理環境を提供し、ユーザーのワークロードに代わってコンピューティング、ネットワーク、ストレージ インフラストラクチャのオーケストレーションを行います。これにより、PaaS (Platform as a Service) のシンプルさと IaaS (Infrastructure as a Service) の柔軟性がもたらされ、インフラストラクチャ プロバイダー間での移植性が向上します。

CircleCI と Kubernetes はどのように連携するか?

CircleCI では、Orb を使用してパイプライン内の Kubernetes サービスを操作できます。Orbs は、再利用可能なコマンド、Executors、ジョブを定義する、パラメーター化された CircleCI 構成のパッケージです。これまで以上に簡単に Kubernetes と CircleCI を組み合わせて利用できるよう、CircleCI チームやパートナーが事前構築済みの Kubernetes Orb のカタログを作成しています。
